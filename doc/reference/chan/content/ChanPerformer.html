Class Overview

Provides performing connectivity with chan.

By default you must implement the following methods:

  • performReadThreads(ReadThreadsData)
  • performReadPosts(ReadPostsData)
  • performReadBoards(ReadBoardsData)

Depending one the ChanConfiguration.Board configuration you must implement the following methods:

OptionMethod
allowSearch performReadSearchPosts(ReadSearchPostsData)
allowCatalog performReadThreads(ReadThreadsData) (see description)
allowArchive performReadThreadSummaries(ReadThreadSummariesData)
allowPosting performReadCaptcha(ReadCaptchaData)
allowPosting performSendPost(SendPostData)
allowDeleting performSendDeletePosts(SendDeletePostsData)
allowReporting performSendReportPosts(SendReportPostsData)

If you configure some options, you also must implement the following methods:

OptionMethod
OPTION_READ_SINGLE_POST performReadSinglePost(ReadSinglePostData)
OPTION_READ_POSTS_COUNT performReadPostsCount(ReadPostsCountData)
OPTION_READ_USER_BOARDS performReadUserBoards(ReadUserBoardsData)
OPTION_READ_POPULAR_THREADS performReadThreadSummaries(ReadThreadSummariesData)
OPTION_ALLOW_CAPTCHA_PASS performCheckAuthorization(CheckAuthorizationData)
OPTION_ALLOW_USER_AUTHORIZATION performCheckAuthorization(CheckAuthorizationData)

public ChanPerformer.ReadCaptchaResult performReadCaptcha (ChanPerformer.ReadCaptchaData data)

Calls when application requests chan to read captcha.

This method must be overridden if option allowPosting is enabled for this board.

This method must return ChanPerformer.ReadCaptchaResult with captcha data.

If your chan uses custom captcha, the result must hold image.

In the case of Yandex Captcha, resulting captchaData must contain challenge string by CHALLENGE. key

In the case of Google reCAPTCHA and Mail.Ru Nocaptcha, resulting captchaData must contain API key by API_KEY. key

If your captcha has short lifetime, you can check mayShowLoadButton. If this argument equals true, the result may hold NEED_LOAD. This will show "Click to load captcha" button for user.
